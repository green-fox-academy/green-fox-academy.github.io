Operators 2

Comparison

When writing code, especially algorithms, you may want your program to behave differently based on your variables' values. Comparison operators exist so that your code can make the correct decision about what to do next. Let's briefly look at the most common ones, and then we'll try them out.

< and <=

The less than < and less than or equal to <= operators let you compare two values to see if the one on the left is less than (or equal) to the one on the right. Take this example:

7 < 7

If you were reading this code aloud to a friend in English, you would say, "7 is less than 7." If your friend disagrees, they would say, "no" (which is correct, 7 is not less than 7).

However, if we say instead, "7 is less than or equal to 7" (written in Java as 7 <= 7), the answer is yes, because 7 is equal to 7.

> and >=

The greater than > and greater than or equal to >= operators can be used the same way as the less than (or equal to) operators. Remember, the statements are always evaluated from left to right, so be mindful of which side your values are on.

== and !=

Since the equality sign = is used for assignment, we need to use the equal to == operator to check for equality. When we want to check if two values are not equal, we need to use the not equal != operator. Again, statements using these operators are read from left to right and you can use them similarly to the other operators addressed in this section.

Now let's try them out!

  1. Is number1 equal to number2?
  2. number2 greater than or equal to number1.
  3. Is number1 less than 5?
  4. number2 not equal to 11.
