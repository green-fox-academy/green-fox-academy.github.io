Syntax and Styling

What is syntax?

In Java, the syntax is the rules for how to write your code so that computers can understand and successfully run it. These include how to correctly structure your code, and what keywords and symbols are available for you to use in different parts of your program. Spelling is also an important part of syntax, especially when using reserved keywords!

You will see a window like this in the following sections:

class Main {
  public static void main(String args[]) {
      // ...
  }
}

Let's take a closer look at the content so that you can understand what the syntax means and why it is important:

class Main {
  // ...
}

This is a class declaration. Classes in Java are blueprints for how to create objects, which is another important Object-oriented programming (OOP) concept. We will cover both in much more detail during the course, so for this week, just know that all the Java code you write needs to be inside of a class's curly braces { } in order for it to run properly. At the start of the line you can see one reserved keyword, class. This is used to create a class. If you spell it cless or clas, your code won't run. The second word, Main, is the name of the class. All classes must have a name. Main is the default when creating new projects, but you can give your classes any descriptive name you want. It is important to remember that in Java, class names must match the name of the file!

class Main {
  public static void main(Strings[] args) {
    // ...
  }
}

Inside of the Main class we now have a method called main. Methods in OOP are blocks of code that have been given a signature that includes the name, what type of input it accepts, and what it will return. We create methods so we only write code once but can reuse code as many times as needed. Don't worry, we will cover methods in more detail during the course, so for this week, just know that all the code you write needs to be inside of the main method's curly braces { }, which will be inside of the Main class's curly braces. There are several reserved keywords in the signature: public static void. Again, we will learn about these keywords during the course, so just remember to spell them correctly!

class Main {
  public static void main(Strings[] args) {
    // This is a single line comment

    /* This is a multi-line comment!
        All the text here will not be compiled */
  }
}

Finally, inside of the main method there are comments. Comments are used to leave messages in your code for your colleagues, users, or even your future self! If your message is short, you can use // to create a single-line comment. All the text after // will not be compiled. If you have more to say, though, use /* <your comment here> */ to spread the text over several lines.

What is styling?

Style conventions, sometimes called "style guides" or "coding conventions," are guidelines rather than strict rules about how your code should be formatted and written. You can see in the windows above that the class's name, Main, starts with a capital letter, but the method's name, main, starts with a lowercase letter. These are examples of Java style conventions, which all good software developers use style guides to make their code uniform and readable. Your mentors will provide more guidance and advice on creating good coding habits throughout the course, so you will have many opportunities to practice and learn.
